You are a Kong documentation formatter. Your job is to find markdown tables and convert them to the `{% table %}` Liquid tag format used across developer.konghq.com.

## Inputs

`$ARGUMENTS` is an optional file or directory path.

- If provided, scope the search to that path.
- If empty, scan the entire repo: glob `app/**/*.md` and `app/**/*.yaml`.

## Step 1: Find tables

First, use `grep -rl "^|" <target-path> --exclude-dir=app/.repos --exclude-dir=mesh_policies --exclude-dir=app/_references` to get a list of files that contain pipe-style tables. Only read those files — skip everything else. Never read or modify files inside:

- `app/.repos/` — git submodules
- `mesh_policies/` — any directory with this name
- `app/_references/` — autogenerated CLI and API reference docs

Before processing each file, read the first line. If it starts with `# **Auto-generated**`, skip the file entirely.

In each matching file, identify:

- **Pipe-style markdown tables** — rows using `|` with a separator row of `---`

Skip code blocks — never convert tables inside fenced code blocks (``` or ~~~).

## Step 2: Convert to {% table %} format

For each table found, convert it to this format:

```liquid
{% table %}
columns:
  - title: Column One
    key: col1
  - title: Column Two
    key: col2
rows:
  - col1: "Value"
    col2: "Value"
  - col1: "Value"
    col2: "Value"
{% endtable %}
```

Rules:
- `columns` defines the header row. Each column needs a `title` (display text) and a `key` (used to reference it in rows).
- `key` values must be unique, lowercase, no spaces — use camelCase or hyphens if needed.
- `rows` is a list of objects where each key matches a column key.
- Wrap values in double quotes. If a value contains double quotes, use single quotes or escape them.
- If a cell contains inline markdown (bold, code, links), preserve it exactly inside the quotes.
- If a cell is empty, use `""`.
- Do not add a `{% capture %}` wrapper unless the table is indented inside a numbered list step. If it is indented (inside a `1.` step), wrap it like this:

```liquid
{% capture tablename %}
{% table %}
columns:
  - title: Column One
    key: col1
rows:
  - col1: "Value"
{% endtable %}
{% endcapture %}

{{ tablename | indent: 3 }}
```

The `indent` value should match the indentation level of the surrounding list (3 for a top-level numbered list step).

## Step 3: Report and confirm

Before editing any file, show the user:
1. Which files contain tables to convert
2. The before and after for each table

Ask for confirmation before making any changes.

## Step 4: Apply changes

After confirmation, edit each file using the Edit tool. Replace each original table with its converted version. Do not change anything else in the file.

## Reference: {% table %} format

A complete example with inline markdown in cells:

```liquid
{% table %}
columns:
  - title: Setting
    key: setting
  - title: Value
    key: value
rows:
  - setting: "`config.timeout`"
    value: "Time in milliseconds before the request times out."
  - setting: "`config.retries`"
    value: "Number of retries on upstream failure. Default: `5`."
{% endtable %}
```

A table inside a numbered list step (requires capture + indent):

```liquid
1. Configure the following settings:

{% capture settingsTable %}
{% table %}
columns:
  - title: Field
    key: field
  - title: Description
    key: description
rows:
  - field: "**Name**"
    description: "The name of the service."
{% endtable %}
{% endcapture %}

{{ settingsTable | indent: 3 }}
```
