## 2. Upload CA Certificate to {{site.base_gateway}}

You need to apply this configuration so that {{site.base_gateway}} generates a UUID for the Certificate.
You'll need to reference this UUID in your Gateway Service.

{% include how-tos/steps/apply_config.md %}

## 3. Retrieve Certificate ID

Run the following command:

```sh
curl -i -X GET http://localhost:8001/certificates
```
{: data-deployment-topology="on-prem" }

```sh
curl -i -X GET https://$KONNECT_PROXY_URL.api.konghq.com/v2/control-planes/$KONNECT_CP_NAME/core-entities/certificates/ \
    --header "Authorization: Bearer $KONNECT_TOKEN" \
```
{: data-deployment-topology="konnect" }

Copy the `id` of your Certificate from the output.

## 3. Create Gateway Service and Route

Create an HTTPS Gateway Service and Route, and reference the CA Certificate by its ID.

Add the following to your `kong.yaml` file:

{% entity_examples %}
entities:
  services:
    - name: example_service
      host: https://self-signed-cert.httpbun.com
      protocol: https
      port: 443
      certificates:
      - 4ae235fd-bcc0-4d08-9689-5ade8f143hjf7
      tls_verify: true
      routes:
        - name: example_route
          paths:
            - /example
{% endentity_examples %}

## 4. Apply configuration

{% include how-tos/steps/apply_config.md %}

## 5. Validate 

Now, any time a request attempts to access the `example_service` through any Route, 
{{site.base_gateway}} will send the client certificate and sign the TLS handshake messages with the corresponding private key.
