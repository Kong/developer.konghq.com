In this guide you'll stand up a local {{site.base_gateway}} instance in DB-less mode, define a Service and Route, and verify that traffic is proxied correctly. You'll then attach the Rate Limiting plugin to see the plugin system in action.

Prerequisites

  • Docker installed and running
  • decK CLI installed (deck version should succeed)

Step 1: Start Kong Gateway in DB-less mode

Run a minimal {{site.base_gateway}} container that reads its configuration from a file:

docker run -d --name kong-gateway \
  -e "KONG_DATABASE=off" \
  -e "KONG_DECLARATIVE_CONFIG=/kong/declarative/kong.yaml" \
  -e "KONG_PROXY_ACCESS_LOG=/dev/stdout" \
  -e "KONG_ADMIN_ACCESS_LOG=/dev/stdout" \
  -e "KONG_PROXY_ERROR_LOG=/dev/stderr" \
  -e "KONG_ADMIN_ERROR_LOG=/dev/stderr" \
  -e "KONG_ADMIN_LISTEN=0.0.0.0:8001" \
  -v "$(pwd)/kong.yaml:/kong/declarative/kong.yaml" \
  -p 8000:8000 \
  -p 8001:8001 \
  kong/kong-gateway:{{site.latest_gateway_oss_version}}

Step 2: Write a declarative configuration

Create a file named kong.yaml in your working directory:

_format_version: "3.0"

services:
  - name: httpbin
    url: https://httpbin.konghq.com
    routes:
      - name: httpbin-route
        paths:
          - /httpbin

This defines one Service (httpbin) pointing to a public echo API, and one Route matching any request whose path starts with /httpbin.

Step 3: Verify the proxy

Send a request through the gateway:

curl -i http://localhost:8000/httpbin/get

You should receive a 200 OK response from httpbin with your request details.

Step 4: Add the Rate Limiting plugin

Update kong.yaml to attach the plugin to the Route:

_format_version: "3.0"

services:
  - name: httpbin
    url: https://httpbin.konghq.com
    routes:
      - name: httpbin-route
        paths:
          - /httpbin
        plugins:
          - name: rate-limiting
            config:
              minute: 5
              policy: local

Reload the configuration using decK:

deck gateway sync kong.yaml --kong-addr http://localhost:8001

Now send more than five requests within a minute. After the fifth, {{site.base_gateway}} returns 429 Too Many Requests — the plugin is active.
