The latest major, minor, and patch versions are made available to you after the Dedicated Cloud Gateway team has tested them.
This is sometimes why a new version is available in {{site.konnect_short_name}} before it's available in Dedicated Cloud Gateways.
All control planes are automatically updated to the latest version after it's available in {{site.konnect_short_name}}.
Dedicated Cloud Gateways use the following upgrade pattern for data plane nodes:
* **Major releases:** ?
* **Minor releases:** Customers are responsible for upgrading their data planes to minor releases following Kong's [upgrade guidance](https://developer.konghq.com/gateway/upgrade/).
* **Patch releases:** Kong upgrades your Dedicated Cloud Gateway data plane nodes for you when a new patch is released.
